GIT

· 2019-07-24 ·

三种状态: 1.已提交(committed) 数据已经安全的保存在本地数据库中 2.已修改(modified) 修改了文件, 还没保存到数据库中 3.已暂存(staged) 都已经修改的文件做了标记, 使之包含在下次提交的快照中

/etc/gitconfig 包含每个用户及仓库的通用配置 ~/.gitconfig或~/.config/git/config 包含个人的配置

.git/config 保存的本仓库的信息

//设置全局信息 只用运行一次 gitconfig−−globaluser.name"xx"git config --global user.name "xx" gitconfig−−globaluser.name"xx"git config --global user.email "xxx"

//设置默认编辑器 $git config --global core.editor vim

//检查配置信息 $git config --list

//查看某一项 $git config user.name

//初始化仓库 在一个新建的空目录下 $git init

//提交修改 gitcommit−m′message′git commit -m 'message' gitcommit−m′message′git commit 会启动编辑器 gitcommit−a自动把所有跟踪过的文件暂存起来一起提交就是跳过gitaddxxxgit commit -a 自动把所有跟踪过的文件暂存起来一起提交 就是跳过git add xxx gitcommit−a自动把所有跟踪过的文件暂存起来一起提交就是跳过gitaddxxxgit commit --amend重新提交

//克隆仓库 $git clone url

//查看当前文件状态 $git status -s/--short

//跟踪一个新文件 $git add filename

//gitignore文件 空行或#开头的会被忽略 可使用glob模式匹配(简化的regex) 匹配以(/)开头防止递归 匹配以(/)结尾指定目录 指定模式以外的文件或目录在模式前加!

//查看不同 $git diff --cached/--staged

//移除文件 gitrmfilename−f(移除已在暂存区的force)git rm filename -f(移除已在暂存区的force) gitrmfilename−f(移除已在暂存区的force)git rm --cached filename 只是不跟踪, 但还在磁盘上

//移动文件 gitmvxxxyyy相当于git mv xxx yyy 相当于 gitmvxxxyyy相当于mv xxx yyy gitrmxxxgit rm xxx gitrmxxxgit add yyy

//查看历史 $git log -p显示差异 -2显示最近两次 --stat简略信息 --pretty=oneline 每个记录只显示一行 --pretty=format:"fmt" --graph

//取消暂存 $git reset HEAD <--hard!!!!!!!!!!!!!!>

远程仓库 $git remote 查看远程仓库 -v查看 k v

添加远程仓库 $git remote add 使用sn 代替 url

//从远程仓库抓取 $git fetch [remote-name]

//推送 remote-name branch-name $git push origin master

//查看远程仓库 $git remote show [remote-name]
